Making a Chamoru Dictionary

By David Taitingfong | September 26, 2021 | 4 minutes
fina'tinas

H친fa adai todus hamyu guennao huyung ni' tumataitai esti!

It’s been over a month-and-a-half since I’ve blogged and not a week has gone by where I thought to myself, “ai adai, debi di bai fa’tinas otru na blog”, but honestly my schedule has just been todu machalapon. I had to take a step back from dang near everything and reasses my priorities and how I spend my time outside of work - hence my absence 游땐.

BUT it was much needed as I’m here now and ready to share a bit about what I’ve been working on, which is: a personal Chamoru dictionary stored on my computer. This project was born out of three “issues” I kept running into:

  1. My favorite go-to resource for looking up Chamoru words (LearningChamoru.com) required too many steps*
  2. There were a lot of words I couldn’t find when searching
  3. Number 1 seems to only reflect the Guam Orthography, and I wanted something “more complete” (i.e., something with words spelled using the CNMI orthography)

* Too many steps = opening a browser, clicking login, logging in, scrolling down to “Dictionary”, and then searching for a word.

Pues, hu tutuhun i hinanao-hu. I created a template of what I thought a dictionary would look like on the back-end (in a database) and I started gathering as many words as I could find. Anytime I came across a new word, I’d pop open my database and add it in there. It took an egregious amount of time but I finally got to a place where I felt comfortable enough to start working on a “front-end” to access the database. I put that in quotes because in it’s current state I can only access it from a terminal/command-line interface (CLI). I started it off this way sa' gi minag친het, it’s where I live when I code. Also it’s the fastest way I can access the database. Essentially I created a small program that allows me to access it via the CLI.

Here’s a screenshot of me running the program:

If you’re unfamiliar with what you’re looking at, it’s a terminal/CLI. The points of interest are the second line, where you see 較덕 $ sodda, and then the text afterwards. My program runs when I type sodda and hit enter. I get prompted for a word to search for (in this case I entered " tungo' “), and then if it finds something I get results back. Otherwise it says it can’t find the word and the prompts just continue until I exit.

Cleaning the data and structuring how everything is stored/returned actually took the most amount of time. I wrote a lot of small helper programs to do things like sort the words, remove misformed words, and add new words, but I still have a lot of work ahead of me. There are still ~3000 words I have that I need to add but don’t have exact translations for. There are also a bunch of words I just straight up don’t have either.

To give you a sense of how big this dictionary has gotten, here is where I’m at:

Currently I store everything in a JSON file, where each word will take up roughly ~20-25 lines. This is near the bottom of the file and as you can see it’s around 340,000 lines (last I checked, there were ~17,000 words). Some functionality is still missing like a) being able to give an English word and get a Chamoru word back, and b) giving suggestions/alternative spellings to what you provided.

Hinasso-ku na sen pairi esti!…lao…there’s one potential problem. I haven’t put this out there yet for everyone to see and try because I actually don’t know if I legally can 游뱂.

I haven’t mentioned this yet, and perhaps you were able to pick it up from earlier when I mentioned why I created this tool, but…I didn’t create a majority of this information from scratch. I made use of LearningChamoru.com, news articles from the GuamPDN, and any blog I could find that seemed like a great resource for filling up a dictionary. So for example, let’s say you use this tool and look up the word “magof” - one of the definitions you’d get back is the same one you’d find in LearningChamoru.com, word-for-word.

Pues hunggan, as much as I want to share this right now, there are still some things I need to figure out. Things like: if I put this dictionary out there, say on a website, would I receive a cease & desist? Could I get sued for copyright infringement? Would I be fine so long as I attribute my data to their correct creators? Etc…

Now that I think about it though…the front-end stuff I can probably share. You’d of course need a dictionary stored in a same-structured JSON, but that general structure I can also share. But for now, this is all I’ll share. When I get more answers and more data and create something more accessible, I’ll give another update!
